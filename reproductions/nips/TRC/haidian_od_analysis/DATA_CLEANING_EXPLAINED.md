# 数据清洗详细说明

## 清洗流程总览

```
原始数据 → 移除空值 → 坐标范围过滤 → 去重 → 清洗完成
```

---

## 步骤1：移除空值（Remove Null Values）

### 做什么？
检查并删除关键字段为空（NULL/NaN）的记录。

**检查的字段：**
- `taxi_id`: 车辆编号
- `date_time`: 时间戳
- `longitude`: 经度
- `latitude`: 纬度

### 为什么这么做？

**1. 数据完整性要求**
```python
# 示例：有问题的记录
taxi_id,date_time,longitude,latitude
1,2008-02-02 15:46:08,116.511,39.938   # ✓ 完整
2,2008-02-02 15:47:00,,39.940          # ✗ 缺少经度
,2008-02-02 15:48:00,116.512,39.941    # ✗ 缺少车辆ID
3,2008-02-02 15:49:00,116.513,         # ✗ 缺少纬度
```

**2. 后续处理必需**
- **空间映射需要经纬度**：如果没有坐标，无法判断点在哪个区域
- **时间映射需要时间戳**：没有时间，无法分配到时间槽
- **出行提取需要车辆ID**：没有ID，无法按车辆分组轨迹

**3. 避免错误传播**
```
空值 → 空间映射失败 → region_id为空 → OD矩阵生成错误 → 结果不可用
```

### 实际影响
```
原始数据: 10,000,000 条
移除空值后: 9,995,000 条
移除比例: 0.05% (通常很小)
```

**如果空值比例很高（>5%）**，说明数据质量有问题，需要：
- 检查数据采集设备
- 检查数据传输过程
- 考虑数据源的可靠性

---

## 步骤2：坐标范围过滤（Coordinate Range Filter）

### 做什么？
只保留在指定地理范围内的GPS点。

**当前设置：**
- 经度范围：116.0° - 116.5°E
- 纬度范围：39.9° - 40.2°N

### 为什么这么做？

**1. 研究区域聚焦**

海淀区的实际范围：
- 经度：约 116.05° - 116.38°
- 纬度：约 39.88° - 40.15°

我们设置的范围稍大（加了缓冲区），目的是：
```
海淀区边界 + 周边缓冲区 = 过滤范围
```

**为什么要缓冲区？**
- **GPS误差**：GPS定位可能有10-50米误差，边界附近的点可能漂移到外面
- **跨边界出行**：车辆从海淀出发，可能开到相邻的朝阳区、西城区
- **避免截断**：如果范围太严格，会把边界的真实数据当作异常删除

**2. 排除明显错误数据**

```python
# 示例：异常坐标
taxi_id,date_time,longitude,latitude
1,2008-02-02 10:00:00,116.30,39.95    # ✓ 海淀区正常点
2,2008-02-02 10:01:00,121.50,31.23    # ✗ 上海的坐标(GPS跳变)
3,2008-02-02 10:02:00,0.00,0.00       # ✗ 无效坐标(设备故障)
4,2008-02-02 10:03:00,116.30,90.00    # ✗ 纬度超出地球范围
```

**常见异常原因：**
- GPS模块故障或重启（返回(0,0)或上次缓存的位置）
- 卫星信号弱导致定位漂移
- 数据传输错误
- 设备时钟错误导致坐标混乱

**3. 提高计算效率**

北京全市数据 vs 海淀区数据：
```
全北京: ~16,000 km²
海淀区: ~431 km²
比例: 海淀只占北京的2.7%

如果不过滤:
- 处理10,000,000条全市数据
- 其中只有~270,000条在海淀区
- 浪费97%的计算资源处理不相关数据
```

**4. 减少噪声**

```
过滤前: 包含全北京甚至其他城市的数据
过滤后: 只保留海淀及临近区域
结果: OD矩阵更清晰，反映海淀区真实流动模式
```

### 实际影响

```
示例数据:
移除空值后: 9,995,000 条
坐标过滤后: 2,257,197 条
保留比例: 22.6%

解释: 约77%的轨迹点不在海淀区范围内（这是正常的）
```

## 步骤3：去重（Remove Duplicates）

### 做什么？
删除完全相同的重复记录。

**判断依据（4个字段完全相同）：**
```python
['taxi_id', 'date_time', 'longitude', 'latitude']
```

### 为什么会有重复？

**1. 数据采集重复**
```
时间    车辆  经度      纬度
10:00   1    116.30   39.95   # 第1次记录
10:00   1    116.30   39.95   # 第2次记录（重复）
```

可能原因：
- GPS设备发送了两次相同的数据包
- 数据库插入时出现重复
- 数据合并时同一份数据被导入多次

**2. 停车等待**

这**不算**真正的重复：
```
时间    车辆  经度      纬度      说明
10:00   1    116.30   39.95   # 停在红绿灯
10:01   1    116.30   39.95   # 还在等待（位置未变）
10:02   1    116.30   39.95   # 继续等待
```

这种情况：
- 时间不同 → 不会被删除 ✓
- 反映真实的停留行为
- 对后续分析有价值

**3. 数据传输错误**

网络传输时，同一条数据可能被发送多次：
```
设备 → 网络 → 服务器
        ↓
      重传机制（如果超时）
```

### 为什么要去重？

**1. 避免重复计数**

```python
# 有重复时:
同一个GPS点被记录3次
→ 在区域统计中被计数3次
→ OD流量虚高3倍
→ 结果失真

# 去重后:
每个真实GPS点只计数1次
→ 统计准确
```

**2. 保证数据唯一性**

```
出行提取逻辑:
车辆1: A点(10:00) → B点(10:10) → C点(10:20)

如果B点重复3次:
错误提取: A→B, B→B, B→B, B→C (4次出行)
正确提取: A→B, B→C (2次出行)
```

**3. 节省存储和计算**

```
示例:
去重前: 2,257,197 条记录
去重后: 2,250,000 条记录
重复率: 0.3% (7,197条重复)

虽然比例不高，但在大数据场景下:
- 减少存储空间
- 加快后续处理速度
- 提高查询效率
```

### 去重的注意事项

**保留哪一条？**

pandas的`drop_duplicates()`默认保留第一条：
```python
# 重复记录
记录1: taxi_id=1, time=10:00, lon=116.3, lat=39.9
记录2: taxi_id=1, time=10:00, lon=116.3, lat=39.9 (重复)
记录3: taxi_id=1, time=10:00, lon=116.3, lat=39.9 (重复)

结果: 保留记录1，删除记录2和3
```

**不会误删的情况：**

1. **不同车辆，相同位置和时间**
```
taxi_id=1, time=10:00, lon=116.3, lat=39.9  # 车1
taxi_id=2, time=10:00, lon=116.3, lat=39.9  # 车2，不同车，保留
```

2. **相同车辆，不同时间**
```
taxi_id=1, time=10:00, lon=116.3, lat=39.9  # 时刻1
taxi_id=1, time=10:01, lon=116.3, lat=39.9  # 时刻2，保留（停车）
```

3. **相同车辆和时间，不同位置**
```
taxi_id=1, time=10:00, lon=116.30, lat=39.9  # 位置1
taxi_id=1, time=10:00, lon=116.31, lat=39.9  # 位置2，保留（虽然异常）
```

## 步骤4：移除异常速度点（可选，当前未启用）

### 做什么？
计算相邻GPS点之间的移动速度，删除不合理的高速点。

```python
速度 = 距离 / 时间差

if 速度 > 阈值（如120 km/h）:
    标记为异常
```

### 为什么需要？

**1. GPS跳变**
```
时间    位置
10:00   海淀区 (116.30, 39.95)
10:01   朝阳区 (116.50, 39.95)  # 瞬间"移动"20公里
10:02   海淀区 (116.30, 39.95)  # 又跳回来
```

计算速度：
```
距离 ≈ 20 km
时间 = 1 分钟
速度 = 1200 km/h ❌ 不可能！
```

**2. 数据错误**
- 时间戳错误（时钟不同步）
- 坐标小数点位移
- 数据库字段错位

### 为什么当前未启用？

**1. 出租车的特殊性**

出租车有合理的高速行为：
- 高速公路行驶：80-120 km/h
- 空载快速返回市区
- 跨区长途送客

**2. 阈值难以设定**

```
太低（50 km/h）: 会误删高速公路数据
太高（150 km/h）: 无法过滤真正的异常
```

**3. 后续过滤会处理**

即使保留异常速度点：
- 如果跳到了海淀区外 → 被坐标过滤删除
- 如果跳到了奇怪的位置 → 空间映射时无法匹配区域 → 被过滤

**4. 需要额外计算**

计算速度需要：
```python
for i in range(len(points)-1):
    distance = haversine(point[i], point[i+1])
    time_diff = point[i+1].time - point[i].time
    speed = distance / time_diff
    # 增加计算复杂度
```

### 如果要启用

可以添加：
```python
def remove_high_speed_points(df, speed_threshold=120):
    """
    移除异常高速点
    speed_threshold: km/h
    """
    df = df.sort_values(['taxi_id', 'date_time'])
    
    # 计算相邻点距离（haversine公式）
    df['distance_km'] = calculate_distance(df)
    
    # 计算时间差（秒）
    df['time_diff_s'] = df.groupby('taxi_id')['date_time'].diff().dt.total_seconds()
    
    # 计算速度（km/h）
    df['speed_kmh'] = (df['distance_km'] / df['time_diff_s']) * 3600
    
    # 过滤异常速度
    df = df[df['speed_kmh'] <= speed_threshold]
    
    return df
```

## 清洗参数调整建议

**当前：**
```python
longitude: 116.0 - 116.5

```

**如何调整：**

| 目标 | 经度范围 | 纬度范围 | 说明 |
|------|----------|----------|------|
| 仅海淀区 | 116.05-116.38 | 39.88-40.15 | 严格边界 |
| 海淀+小缓冲 | 116.0-116.5 | 39.9-40.2 | 当前设置（推荐） |

# 更严格：只要ID和时间相同就去重

## 总结

| 步骤 | 目的 | 效果 | 必要性 |
|------|------|------|--------|
| 移除空值 | 保证数据完整 | 删除0.05% | ★★★★★ 必需 |
| 坐标过滤 | 聚焦研究区域 | 删除77% | ★★★★★ 必需 |
| 去重 | 避免重复计数 | 删除0.3% | ★★★★☆ 强烈推荐 |
| 速度过滤 | 删除异常跳变 | 未启用 | ★★☆☆☆ 可选 |

**核心原则：**
1. **宁可少删不多删** - 避免误删真实数据
2. **先粗后细** - 先用明显规则过滤，后续可再细化
3. **可追溯** - 每步都记录删除了多少，保证透明
4. **可调整** - 参数化设计，方便根据实际情况调整
