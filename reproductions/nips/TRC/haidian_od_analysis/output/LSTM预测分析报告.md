## 5️⃣ LSTM预测方法详解

### 🧠 模型架构

```
输入层 (12个时间槽 × 739个OD对)
    ↓
LSTM Layer 1 (64 hidden units)
    ↓
LSTM Layer 2 (64 hidden units)
    ↓
Dropout (0.2)
    ↓
全连接层 1 (64 → 32)
    ↓
ReLU激活
    ↓
全连接层 2 (32 → 739)
    ↓
输出层 (预测下1个时间槽的739个OD流量)
```

**总参数量**: 265,827

### 🔄 预测流程

1. **数据准备**
   - 输入: 前12个时间槽的OD流量矩阵 (96 × 29 × 29)
   - 展平为: (12, 739) - 12个时间步，739个活跃OD对
   - 归一化: MinMax缩放到[0, 1]

2. **滑动窗口预测**
   ```
   时间槽 0-11  → 预测 时间槽 12
   时间槽 1-12  → 预测 时间槽 13
   时间槽 2-13  → 预测 时间槽 14
   ...
   时间槽 83-94 → 预测 时间槽 95
   ```

3. **训练策略**
   - 训练集: 前80%样本 (67个样本)
   - 测试集: 后20%样本 (17个样本)
   - 优化器: Adam (学习率 0.001)
   - 损失函数: MSE (均方误差)
   - 早停: 连续15个epoch无改进则停止
   - 实际训练: 33个epoch

4. **预测生成**
   - 对每个时间槽预测739个OD对的流量
   - 反归一化还原真实值
   - 过滤负值，确保流量≥0
   - 生成CSV格式输出

### ⚙️ 关键技术特点

| 特性 | 说明 |
|------|------|
| **序列长度** | 12个时间槽 (3小时历史数据) |
| **预测步长** | 1个时间槽 (15分钟ahead) |
| **多变量预测** | 同时预测739个OD对 |
| **归一化** | MinMaxScaler防止数值不稳定 |
| **正则化** | Dropout(0.2)防止过拟合 |
| **梯度裁剪** | max_norm=1.0防止梯度爆炸 |
| **学习率调度** | ReduceLROnPlateau自适应降低 |
| **设备加速** | GPU (CUDA)加速训练 |

## 7️⃣ 优势与局限

### ✅ 优势

1. **格式完全兼容**: 输出与原始CSV格式一致
2. **端到端预测**: 自动处理所有活跃OD对
3. **时序建模**: LSTM捕捉时间依赖关系
4. **训练/测试分离**: 避免数据泄露
5. **高峰时段准确**: 流量高峰预测误差<10%
6. **可扩展性**: 支持更多天数据训练

### ⚠️ 局限

1. **短序列限制**: 只有1天数据，训练样本少(84个)
2. **冷启动问题**: 前12个时间槽无法预测
3. **小流量噪声**: 低流量OD对MAPE高
4. **周期性缺失**: 无法学习周/月模式
5. **静态特征**: 未融合天气、节假日等外部因素
